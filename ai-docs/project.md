# Project Context

Project:
Parcels & Zoning Data Pipeline

Purpose:

Build a small data product capable of ingesting two ArcGIS Feature Services, normalizing them into PostgreSQL/PostGIS and exposing REST APIs for spatial queries.

This project follows Spec Driven Development.

Implementation must always be driven by an approved specification before code is written.

Priority:

1. Correctness
2. Simplicity
3. Maintainability
4. Performance

Never optimize prematurely.

Time budget:

Approximately four hours.
